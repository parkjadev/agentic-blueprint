# Spec-author assets

Pre-stripped copies of spec templates for rapid `docs/specs/<slug>/` creation.

At runtime, the `spec-author` skill uses the canonical templates in `docs/templates/` via `Read`. These asset copies are not the source of truth — they exist only as a fallback if a subagent needs a template and `docs/templates/` is unavailable.

**If you find yourself reaching for these assets, read `docs/templates/` instead.** They're always authoritative.
